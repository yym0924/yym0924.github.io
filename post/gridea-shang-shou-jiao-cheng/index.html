Gridea上手教程

2019-06-29

使用Gridea搭建和管理GitHub Pages

本地软件安装

Git 安装

Gridea的同步功能依赖本地Git客户端的，所以我们需要先安装Git。 Git官网下载平台对应的安装程序。安装没有特别的要求，一路下一步即可。

Gridea 安装

Gridea官网下载对应平台的安装程序进行安装。

初始配置

GitHub Pages 创建

仓库创建

首先，需要创建一个仓库，建议仓库为 username.github.io 。根据官方文档， Github 会默认将此仓库设置成 GitHub Pages 可访问类型，否则不能默认使用 Github Pages。例如我的用户名是 yym0924 ，所以仓库名就是 yym0924.github.io 。不过我自己创建仓库的时候遇到并未自动启用 GitHub Pages ，遇到这种情况手动开启即可。

基本设置

开通GitHub Pages

按照官方主页的描述，仓库名符合 username.github.io 的新建仓库都会自动启用GitHub Pages。但是我创建这个博客的时候并没有自动启用，所以这里也说明下手动启用的方式。 打开仓库的 Settings

找到 GitHub Pages 设置的地方 点击 Choose a theme 随意选择一个主题，因为后续我们使用Gridea设置主题，这边选择主题只是为了开启GitHub Pages可访问类型。

创建新token

Gridea 管理GitHub需要token。登录GitHub后点击这里 创建一个新的token。 勾选上 repo 相关权限即可。

请把新建的token复制一个地方保存起来，因为关闭这个网页后你就再也看不到它了。

Gridea初始配置

做好基本设置并保存，这步设置好后就可以开始使用。你可以点击客户端左下角的预览按钮来查看本地渲染效果，或者点击同步进行博客最新内容的部署。

开启文章评论

Gridea支持 Gitalk 和 Disqus 两种评论系统，这里我使用Gitalk评论系统。

Gitalk 是一个基于 GitHub Issue 和 Preact 开发的评论插件。 首先你需要有一个 Github OAuth Applicatio ，点击这里创建。 创建完成后可将 Client ID 和 Client Secret 复制保存到本地。 设置、保存并同步后评论模块就可用了。
